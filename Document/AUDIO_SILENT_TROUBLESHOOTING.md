# 音频播放无声问题排查清单

## 问题现象
- I2S 初始化成功，无错误日志
- 音频播放开始日志正常
- 但喇叭没有声音输出

## 硬件检查

### 1. MAX98357A 模块接线检查 ✅

#### 必须连接的引脚：
| MAX98357A 引脚 | ESP32-S3 引脚 | 状态 | 说明 |
|---------------|--------------|------|------|
| **LRC** | GPIO21 | □ | 左右声道时钟（字选择） |
| **BCLK** | GPIO47 | □ | 位时钟 |
| **DIN** | GPIO48 | □ | 数据输入 |
| **GND** | GND | □ | 接地 |
| **VIN** | 5V | □ | 电源正极 |
| **SD** | GND | □ | **关键！必须接 GND 才能使用 I2S 模式** |
| **GAIN** | 悬空 | □ | 增益设置（默认 -3dB） |
| **OUT+** | 喇叭+ | □ | 音频输出正极 |
| **OUT-** | 喇叭- | □ | 音频输出负极 |

#### ⚠️ 关键检查点：

**SD 引脚必须接 GND！**
- SD 引脚决定工作模式：
  - 接 GND → I2S 模式（正确）✅
  - 接 VCC → 左/右声道混合模式 ❌
  - 悬空 → 不确定状态 ❌

**检查方法**：
1. 用万用表测量 SD 引脚与 GND 之间的电阻
2. 应该接近 0 欧姆（导通）
3. 如果不导通，请用杜邦线将 SD 引脚连接到 GND

### 2. 喇叭连接检查

#### 喇叭类型：
- **有源喇叭**：需要外部电源，信号线接 ESP32
- **无源喇叭**：直接接 MAX98357A 输出端

#### 正确连接方式：
```
MAX98357A        喇叭
---------        ----
OUT+    --->     + (正极)
OUT-    --->     - (负极)
```

#### 检查方法：
1. 确认喇叭正负极连接正确
2. 用万用表测量喇叭阻抗（应该 4-8 欧姆）
3. 确认喇叭本身没有损坏

### 3. 电源检查

#### MAX98357A 供电：
- **VIN**: 5V 电源（从 ESP32 的 5V 引脚或外部 5V 电源）
- **GND**: 接地

#### 检查方法：
1. 用万用表测量 VIN 和 GND 之间的电压
2. 应该在 4.5V - 5.5V 之间
3. 如果电压过低，检查电源连接

### 4. I2S 信号检查（可选）

#### 使用示波器或逻辑分析仪：
1. 测量 BCLK 引脚：应该有 ~3MHz 的时钟信号（44.1kHz × 32bit）
2. 测量 LRC 引脚：应该有 ~44.1kHz 的方波
3. 测量 DIN 引脚：应该有数据信号

#### 使用万用表（简易方法）：
1. 测量 BCLK 对地电压：应该在 1.5V - 2.5V 之间（不是 0V 或 3.3V）
2. 测量 LRC 对地电压：应该在 1.5V - 2.5V 之间
3. 如果电压为 0V 或 3.3V，说明引脚没有输出信号

## 软件调试

### 1. 查看串口日志

#### 正常播放日志：
```
[NetworkAudioPlayer] 播放音频 URL: https://...
[NetworkAudioPlayer] 开始播放...
[NetworkAudioPlayer] 音频解码器状态：运行中
[NetworkAudioPlayer] 播放已开始
[NetworkAudioPlayer] 播放状态检查 - 运行中：是
[NetworkAudioPlayer] 当前音量：20
```

#### 异常日志：
```
[NetworkAudioPlayer] 连接失败！  // 网络问题
[NetworkAudioPlayer] 音频解码器状态：未运行  // 解码失败
[NetworkAudioPlayer] 播放状态检查 - 运行中：否  // 播放停止
```

### 2. 测试 Speaker 蜂鸣声

**目的**：确认 MAX98357A 和喇叭本身是否正常

**方法**：
1. 按压压力传感器
2. 应该听到 1 秒蜂鸣声
3. 如果蜂鸣声正常，说明硬件连接正确
4. 如果蜂鸣声也没有，说明硬件连接有问题

### 3. 测试网络音频播放

**方法**：
1. 服务器下发带 audio_url 的闹钟任务
2. 等待闹钟触发
3. 观察串口日志
4. 如果日志显示"运行中"但无声，可能是：
   - 音频格式不兼容
   - 音量太小
   - 喇叭损坏

## 常见问题解决

### 问题 1：SD 引脚未接 GND
**现象**：I2S 初始化成功，但无声
**解决**：将 SD 引脚用杜邦线连接到 GND

### 问题 2：喇叭正负极接反
**现象**：声音很小或无声
**解决**：交换喇叭正负极连接

### 问题 3：电源不足
**现象**：声音失真或断断续续
**解决**：使用外部 5V 电源供电

### 问题 4：音频格式不兼容
**现象**：日志显示运行中但无声
**解决**：
- 确认服务器音频是 MP3 格式
- 确认采样率为 44.1kHz 或 48kHz
- 确认比特率在 128kbps 以下

### 问题 5：I2S 端口冲突
**现象**：Speaker 工作但 NetworkAudioPlayer 不工作
**解决**：
- Speaker 使用 I2S_NUM_1
- NetworkAudioPlayer 使用 ESP32-audioI2S 库（内部使用 I2S_NUM_0）
- 两个模块使用不同的 I2S 端口（已修复）
