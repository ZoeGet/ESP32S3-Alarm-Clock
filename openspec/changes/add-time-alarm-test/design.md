## Context

当前系统已实现ESP32S3的实时时间显示和LED闪烁功能，现在需要添加时间到点播报功能。由于硬件限制，只能直接连接8Ω/0.5W喇叭到ESP32S3，无其他元件（如三极管、电阻等），因此需要在确保ESP32安全的前提下实现测试功能。

## Goals / Non-Goals

**Goals:**
- 实现时间到点检测和声音播报功能
- 安全驱动8Ω/0.5W喇叭（低占空比PWM）
- 集成到现有系统架构
- 验证时间触发逻辑的正确性

**Non-Goals:**
- 不追求高音量效果（受硬件限制）
- 不实现复杂的声音模式（仅测试基本蜂鸣）
- 不修改现有时间显示和LED功能
- 不添加新的硬件元件

## Decisions

### 1. 硬件连接方案：直连GPIO + 低占空比PWM

**决策**：使用GPIO25作为PWM输出，直接连接8Ω/0.5W喇叭

**硬件连接：**
```
ESP32 GPIO25 ──────────────── 喇叭正极
GND ─────────────────────── 喇叭负极
```

**理由**：
- GPIO25是通用GPIO，支持PWM功能
- 位置方便，不与现有功能冲突
- 避免使用BOOT、串口等关键引脚

**风险缓解**：使用极低占空比（≤5%），限制电流输出在安全范围内

### 2. 软件架构：模块化设计

**决策**：创建独立的`Speaker`和`Alarm`类，集成到现有系统

**架构图：**
```
┌─────────────────┐
│    main.cpp     │
└────────┬────────┘
         │
┌────────┴────────┬─────────────┬─────────────┐
│    Clock.h     │  OLED_Display.h │   Timer.h   │
└────────┬────────┘               └─────────────┘
         │
┌────────┴────────┐
│   Alarm.h       │
└────────┬────────┘
         │
┌────────┴────────┐
│  Speaker.h      │
└─────────────────┘
```

**模块职责：**
- `Speaker`：PWM驱动，安全控制喇叭
- `Alarm`：时间检测，触发声音播报
- 现有模块：保持不变

### 3. 时间触发逻辑：每秒检测

**决策**：在`loop()`中每秒检测时间是否到达设定值

**逻辑流程：**
```
loop():
  1. 更新LED
  2. 显示时间
  3. 检测是否到达提醒时间
     - 到达：触发声音播报
     - 未到达：继续
  4. 等待下一个周期
```

**时间比较精度：** 精确到秒，满足测试需求

### 4. 声音控制：固定频率蜂鸣

**决策**：使用1kHz固定频率，100ms持续时间

**声音参数：**
- 频率：1kHz（人耳敏感范围）
- 持续时间：100ms（足够识别）
- 占空比：5%（安全电流限制）

**理由**：
- 1kHz声音清晰可辨
- 短持续时间减少ESP32负载
- 低占空比保证安全

### 5. 安全保护：多重保护机制

**决策**：实现硬件和软件双重保护

**保护措施：**
1. **硬件保护**：使用低占空比PWM（≤5%）
2. **软件保护**：
   - 限制单次发声时间（≤200ms）
   - 限制发声频率（最小间隔1秒）
   - 检测PWM通道状态
3. **错误处理**：PWM初始化失败时跳过声音功能

## Risks / Trade-offs

**[风险] ESP32过载** → 缓解：使用极低占空比（5%），限制单次发声时间

**[风险] 声音微弱** → 缓解：作为测试验证，只要能听到即可，不追求音量

**[风险] PWM干扰** → 缓解：使用独立的PWM通道，避免与其他功能冲突

**[风险] 硬件损坏** → 缓解：先进行短时间测试，确认安全后再正常使用

**[权衡] 功能 vs 安全**
- 优先保证ESP32安全，其次实现测试功能
- 接受声音微弱的限制，确保硬件安全

## Migration Plan

1. **添加新模块**：
   - 创建`Speaker.h/cpp`：PWM驱动
   - 创建`Alarm.h/cpp`：时间检测

2. **修改现有模块**：
   - 修改`main.cpp`：集成新功能
   - 修改`OLED_Display.h/cpp`：显示提醒时间设置

3. **测试流程**：
   - 首先进行短时间（10ms）测试
   - 确认硬件安全后，进行完整测试
   - 验证时间触发逻辑

4. **验证标准**：
   - 时间到点时能听到声音
   - ESP32运行稳定
   - 现有功能不受影响
