## Context

当前RTC时钟系统使用固定`delay(1000)`实现每秒刷新，但存在以下问题：

1. **累积误差问题**：每次循环包含代码执行时间（读取RTC、格式化、OLED刷新、LED更新），导致实际周期 > 1000ms
   - 例如：执行耗时15ms → 实际周期1015ms → 每分钟累积误差约1秒
   - 长时间运行后，显示时间与实际时间偏差明显

2. **刷新不同步问题**：屏幕刷新与RTC秒边界不对齐，造成视觉上的"跳秒"现象

当前代码结构：
```cpp
void loop() {
    led.update();      // ~1ms
    displayTime();     // ~10-20ms (包含OLED刷新)
    delay(1000);       // 固定1000ms，未补偿执行时间
}
```

## Goals / Non-Goals

**Goals:**
- 实现精确的1秒刷新周期，消除累积误差
- 使用`millis()`时间戳对齐机制替代固定延迟
- 自动补偿代码执行耗时
- 保持现有RTC、OLED、LED功能不变

**Non-Goals:**
- 不修改RTC初始化逻辑
- 不修改OLED显示格式和位置
- 不修改LED闪烁参数
- 不引入外部RTC硬件（DS3231等）
- 不实现秒边界硬同步（忙等待方案）

## Decisions

### 1. 时间对齐策略：绝对时间戳 vs 相对补偿

**决策**：使用绝对时间戳（`nextUpdateTime`）跟踪下次刷新点

**逻辑流程**：
```
初始化：
  nextUpdateTime = millis() + 1000  // 设置首次刷新时间

主循环：
  1. 执行显示更新（led.update() + displayTime()）
  2. 计算等待时间：waitTime = nextUpdateTime - millis()
  3. 如果 waitTime > 0，延迟 waitTime
  4. 如果 waitTime <= 0（执行超时），立即继续
  5. 更新下次刷新时间：nextUpdateTime += 1000
```

**理由**：
- 自动累积补偿：如果某次执行超时，下次会相应调整，长期保持准确
- 避免误差累积：每次刷新基于绝对时间点，不受单次执行波动影响
- 实现简单：仅需一个全局变量

**替代方案**：相对补偿（`delay(1000 - elapsedTime)`）
- 缺点：每次仍有微小误差，长期仍会累积

### 2. 超时处理策略：忙等待 vs 跳过

**决策**：当执行超时时，不跳过任何刷新，立即继续并更新基准

**逻辑**：
```cpp
if (waitTime > 0) {
    delay(waitTime);  // 正常等待
} else {
    // 执行超时，不等待立即继续
    // 但nextUpdateTime仍按规律递增
}
nextUpdateTime += 1000;  // 始终递增，保持节奏
```

**理由**：
- 保持1秒节奏：即使某次延迟，下次仍按原节奏执行
- 避免追赶效应：不试图"补回"丢失的时间
- 符合时钟显示习惯：用户期望稳定的1秒间隔

**替代方案**：跳过 missed 的刷新
- 缺点：可能造成显示"卡顿"

### 3. 变量作用域：全局变量 vs 局部静态变量

**决策**：使用全局变量`nextUpdateTime`

**理由**：
- `loop()`函数需要跨调用保持状态
- 全局变量语义清晰，易于理解
- 避免静态变量的初始化复杂性

**替代方案**：局部静态变量
- 缺点：首次调用需要特殊初始化逻辑

## Risks / Trade-offs

**[风险] millis()溢出** → 缓解：`millis()`约49.7天溢出一次，使用无符号长整型运算，溢出后计算仍然正确
```cpp
// 无符号运算自动处理溢出
unsigned long waitTime = nextUpdateTime - currentTime;  // 即使溢出，差值仍正确
```

**[风险] 中断影响** → 缓解：`millis()`由定时器中断维护，精度不受主循环影响。如果中断被长时间禁用，可能影响精度，但正常Arduino代码不会这样做。

**[风险] 执行时间波动** → 缓解：OLED刷新时间可能因内容变化而波动（如复杂图形），但当前仅显示时间字符串，执行时间相对稳定（约10-20ms）。

**[权衡] 精度 vs 复杂度**
- 当前方案精度约±1ms，实现简单
- 如需更高精度（微秒级），需要硬件定时器中断，复杂度显著增加
- 对于时钟显示场景，±1ms精度完全足够

## Migration Plan

1. **添加全局变量**：在文件顶部添加`unsigned long nextUpdateTime = 0;`
2. **修改setup()**：在初始化完成后设置`nextUpdateTime = millis() + 1000;`
3. **修改loop()**：
   - 移除`delay(1000)`
   - 添加时间计算和动态延迟逻辑
4. **测试验证**：
   - 观察1分钟，确认秒数变化准确
   - 观察10分钟，确认无累积误差
